# SP2 — Library Rebuild + 3D Viewer: Design

**Status:** approved, ready for plan · **Date:** 2026-07-07
**App:** KiCad Library Manager (PyQt5 desktop, `git/Hardware`)
**Repo (canonical):** `~/git/Hardware` (WSL-native), branch `ui-clean-slate`
**Predecessor:** SP1 — Self-Contained Core (complete)

---

## 0. Where this sits

SP1 made the frozen `.exe` self-contained (bundled DB, baked key, seeded library,
writable location). SP2 is the second of the three sub-projects from the overhaul:

- **SP1 — Self-Contained Core.** Done.
- **SP2 — Library rebuild + 3D viewer (this doc).** Turn the Library page into a
  master–detail workspace, wire in the already-existing `fp_render` symbol / footprint /
  3D renderers, fix the two dead controls, and surface every *Library-scoped* stranded
  backend feature.
- **SP3 — App-wide seamlessness pass.** The remaining stranded logic that lives on
  *other* pages (Bench STM32 exporters, the extended Projects settings layer).

**Two standing requirements (from the overhaul doc §0), enforced in SP2:**
1. **Every UI change is validated by an offscreen screenshot (dark + light) and critiqued
   against `docs/design/design-rules.md`** — the user cannot run the app himself. The bar
   for this sub-project is explicit: **the UI must be pristine with zero critiques.**
2. **The UI must expose the logic that exists.** SP2 wires in all Library-scoped stranded
   logic; §7 lists what is deferred to SP3 and why, so nothing is lost.

---

## 1. Problem

The Library page (`tools/ui/features/library.py`) is a 3-tab `Workspace`
(Parts / Sourcing / Import). Two gaps:

1. **The Parts tab is a dead-end table.** It lists grouped parts (part number,
   manufacturer, three asset flags) with no selection, no detail, and **no previews at
   all** — even though a complete render stack (`fp_render`: `render_symbol_image`,
   `render_footprint_image`, `render_step_image` + interactive `paint_mesh`,
   `footprint_summary`, `step_summary`) is bundled and fully working, just uncalled.
2. **Stranded Library logic + dead controls.** "Enrich From Part Number" (`library.py:71`)
   has no handler; the Projects net-class Profile selector (`projects.py:378`) has no
   `on_change`; and `search_parts`, `enrich_library`/`enrich_symbol`, `export_catalog`,
   `find_corrupt_kicad_files`, and the group-override machinery
   (`load_group_overrides`/`save_group_overrides`) have backends but no UI.

## 2. Goal / non-goals

**Goal:** selecting a part in the Library shows its symbol, footprint, and an interactive
3D model side by side; every Library-scoped backend feature has a control; the two dead
controls work; and every screen passes the design-rules checklist with zero critiques.

**Non-goals (SP2):**
- No changes to how any backend logic *works* — SP2 only *calls* existing pure helpers.
- No new distributor integrations (Mouser only, as SP1 left it).
- No Bench or Projects redesign beyond the single Profile-selector fix (SP3 owns those).
- No new render engine — `fp_render` is used verbatim.

## 3. Locked decisions

| # | Decision | Choice |
|---|---|---|
| 1 | Library layout | **Master list + stacked previews.** Left = selectable parts list; right = detail with Symbol / Footprint / 3D **stacked together** (no preview sub-tabs). Sourcing & Import stay sibling `Workspace` tabs. |
| 2 | 3D viewer | **Interactive** drag-to-orbit + scroll-to-zoom (`paint_mesh`), mirroring Bench's custom-painted widget. Graceful fallback ladder (§5.3). |
| 3 | Dead controls | **Both** fixed: Enrich (Library) and the Projects Profile selector. |
| 4 | Stranded logic | **All Library-scoped features wired** (§6). Bench/Projects-scoped features explicitly deferred to SP3 (§7). |
| 5 | New module | Preview widgets live in a new `tools/ui/features/library_preview.py`; `library.py` stays the feature/orchestration file. |
| 6 | Rendering thread | Renders run **off the GUI thread** via the shell async service; `fp_render`'s built-in image cache is reused; cards show a placeholder until the render arrives. |

## 4. Architecture

The Library page remains `W.Workspace(ctx, "Library", panels)` with the same three tabs.
All new structure is inside the **Parts** panel.

```
Library  [ Parts | Sourcing | Import ]
+--------------------------------------------------------------+
| Library Health  <Verdict band, spans full width>            |
+------------------+-------------------------------------------+
| search...        |  STM32F103C8T6                            |
| F103C8T6      *  |  STMicroelectronics · MCU 32-bit          |
| USB-C 16pin      |  Datasheet ->                             |
| R_0402           |  +--------------------------------------+  |
| 2N7002           |  | Symbol      (render_symbol_image)    |  |
| ...              |  +--------------------------------------+  |
|                  |  | Footprint   (render_footprint_image) |  |
| [Export Catalog] |  |             footprint_summary caption|  |
|                  |  +--------------------------------------+  |
|                  |  | 3D Model    (interactive MeshView)   |  |
|                  |  |             step_summary caption      |  |
|                  |  +--------------------------------------+  |
+------------------+-------------------------------------------+
```

### 4.1 Components

- **`_parts_panel(ctx, state)`** (rewritten, `library.py`) — builds the master–detail
  layout: the `Verdict` health band on top, then a horizontal split of `PartsList`
  (master) and `PartDetail` (detail). Owns the selection callback: `list.on_select(row)
  → detail.show(row)`. First row auto-selected. The "Export Catalog" action sits on the
  master column's command row.
- **`PartsList`** (`library_preview.py`) — a selectable list built from
  `scan_library_grouped(cfg)` rows. Each entry: part label (mpn or name, mono) + a single
  6px asset-completeness dot (complete = neutral text_1; missing model = `warn`; dangling
  = `err`). A search box above filters client-side over name / mpn / manufacturer. No
  borders, full-row hover/selected wash `bg_inset` per design-rules.
- **`PartDetail`** (`library_preview.py`) — a `scroll_body` with an identity header
  (mpn text_1, manufacturer · description text_2 sentence-case, Datasheet link text_2)
  followed by three stacked `PreviewCard`s (Symbol, Footprint, 3D Model). `show(row)`
  swaps content; each card renders independently.
- **`PreviewCard`** (`library_preview.py`) — one `bg_inset` container (8px radius, no
  border, no card-in-card) holding a quiet section eyebrow, the render surface, and an
  optional dim caption (footprint pads/size, model format/bbox). Shows a placeholder
  while the async render is in flight, and a dimmed "No Symbol / No Footprint /
  3D Preview Unavailable" state when the asset is absent or the render returns None.
- **`MeshView(QWidget)`** (`library_preview.py`) — the interactive 3D widget (§5.2).

### 4.2 Data flow (per selected row)

`scan_library_grouped` already returns everything needed per part:
`name, mpn, manufacturer, datasheet, description, footprint (stem), symbols (names),
model (basename), model_source, has_symbol, has_footprint, has_model, dangling`.

On `show(row)`:
- **Symbol** — read `SymbolLib`, find the block for `row["symbols"][0]`
  (`extract_symbol_blocks` + `extract_symbol_name`), pass its text to
  `render_symbol_image(block_text)`.
- **Footprint** — `Path(cfg["FootprintLib"]) / f'{row["footprint"]}.kicad_mod'` →
  `render_footprint_image(path)`; caption from `footprint_summary(path)`.
- **3D** — `Path(cfg["ModelLib"]) / row["model"]` → `MeshView(path)`; caption from
  `step_summary(path)`.

Each render is dispatched through the shell async service (`run_populate`-style) so
parsing/rasterizing never blocks the GUI; results are cached by `fp_render`.

## 5. Detailed behavior

### 5.1 Symbol & footprint previews

Static `QImage`s scaled into the card. Both `render_symbol_image` and
`render_footprint_image` return `Optional[QImage]`; None → the card's dim empty state.
The footprint caption reads pad count and courtyard size from `footprint_summary`; the
symbol card needs no caption (the identity header already names the part).

### 5.2 Interactive 3D viewer (`MeshView`)

Mirrors the Bench interactive-widget pattern (custom `QWidget`, `paintEvent`,
mouse/wheel handlers):

- **Load once** — pick loader by `model_format(path)`: `load_step_mesh` (STEP/STP) or
  `load_vrml_mesh` (WRL). Loading runs off-thread; until `(verts, faces)` arrive the
  widget paints the placeholder.
- **Paint** — `paintEvent` calls `fp_render.paint_mesh(painter, w, h, verts, faces,
  rot_x=self._rx, rot_y=self._ry, zoom=self._zoom)`.
- **Interact** — `mousePressEvent` captures the anchor; `mouseMoveEvent` accumulates
  delta into `_rx`/`_ry` and calls `update()`; `wheelEvent` scales `_zoom` (clamped) and
  calls `update()`. Sensible initial angle (`rot_x=-60, rot_y=-35`, matching
  `render_step_image`).

### 5.3 Fallback ladder (3D)

1. `have_3d()` False (no numpy / mesh backend) **or** mesh load returns None →
2. fall back to static `render_step_image(path)` painted into the card →
3. that also None → dim "3D Preview Unavailable" caption.

`has_model` False (part has no linked model) → the card shows "No 3D Model" without any
load attempt.

### 5.4 Enrich From Part Number (dead control → wired)

Library → Sourcing button. On click:
1. `enrich_library(cfg, lookup, dry_run=True)` off-thread → returns
   `{changes, written, symbols, looked_up}`.
2. Report "N Fields Fillable Across M Parts" in the sourcing result area.
3. If `changes`, offer "Apply" → re-run `enrich_library(..., dry_run=False)` (snapshots to
   `.trash` by construction, fills blanks only). Log via the existing `LogSink`.
Uses the same `lookup = LM.providers_from_config(cfg)` guard the Refresh button uses; the
button is disabled when no key is configured.

### 5.5 Net-class Profile selector (dead control → wired)

Projects → `_netclass_panel`. Wire `W.Segmented(profiles).on_change(fn)` so selecting a
profile:
1. rebuilds the net-class grid from `create_vault_standard_template(profile)`, and
2. re-runs `validate_netclasses(mgr, profile)` against the selected profile.
The grid rebuild reuses the existing grid-population code path (extracted to a small
`_populate_grid(mgr)` helper so the selector and initial build share it).

## 8. Design-rules compliance (the pristine bar)

The new surfaces must pass `docs/design/design-rules.md` with **zero** critiques.
Specific commitments, mapped to the rules:

- **One elevation per region** (rule 3): the detail pane is `bg_raised`; each
  `PreviewCard` is the single `bg_inset` step. No card-in-card — the render sits directly
  on the inset surface, not in a nested outlined box.
- **Zero borders** (rule 2): regions separated by whitespace + the elevation step. The
  only hairlines are the section eyebrow trailing rules and the one master/detail divide.
- **Color is meaning** (rules 5, 6): the asset-completeness dot is the only color in the
  list (semantic `ok`/`warn`/`err` at 6px, never a surface tint, never an accent bar).
  Selection uses the azure wash, not a category hue.
- **One focal point** (rule 7): in the detail pane the identity header (part number) is
  dominant; captions and eyebrows recede to text_3.
- **No pills on data** (rule 1): part numbers are mono text with hierarchy, not chips.
- **Title Case UI text** (§2): "Export Catalog", "Search Mouser", "Enrich From Part
  Number", "3D Model", "Re-link"; sentence case only for status/empty-state sentences.
- **Radius** (rule 9): 8px for the preview containers, 6px for controls/hover — the two
  locked values, nothing else.

## 9. Error handling

- Every `fp_render` call already returns `Optional`; a failed/timed-out render degrades to
  the card's dim empty state, never a crash or a traceback in the UI.
- Selection is read-only — nothing mutates when a part is picked, so rapid selection is
  safe. The only mutating actions (Enrich apply, Re-link, Export) go through the existing
  async/log/snapshot machinery.
- A missing library path (`SymbolLib`/`FootprintLib`/`ModelLib` absent) shows the existing
  "Library Unavailable" panel, unchanged.

## 10. Testing & validation

**Unit (pytest, headless):**
- Selection → preview plumbing: for a given grouped row, the correct symbol block text,
  footprint path, and model path are chosen (including the ungrouped / missing-asset
  rows).
- Fallback ladder: `have_3d()` False → static path taken; static None → caption state.
- Enrich flow: dry-run returns a change set without writing; apply writes and snapshots.
- Profile selector: `on_change` rebuilds the grid from the selected profile's template and
  validates against it.
- Stranded wiring: each new control invokes the right backend with the right args (mocked
  lookups where network would be involved).

**Offscreen render validation (mandatory — dark + light):**
- Parts master–detail with a real seeded part: all three previews populated.
- Empty / fallback states (no model; render None).
- Critique each against `docs/design/design-rules.md` §5 checklist; **zero critiques is
  the ship gate** for this sub-project.

Render recipe (from SP1): `QT_QPA_PLATFORM=offscreen`, `T.set_theme(dark)`,
`app.setStyleSheet(T.qss(dark))`, `W.restyle_all()`, `panel.grab().save(...)`, then repeat
for light.

## 12. Risks & mitigations

| Risk | Mitigation |
|---|---|
| Mesh render blocks the GUI on a big STEP | Off-thread load + `fp_render` cache; placeholder until ready. |
| `paint_mesh` interactive frames look wrong and can't be tested live | Validate several offscreen angles; keep the static fallback as the safety net. |
| New surfaces reintroduce "AI-made" texture (borders/pills/tints) | Hard design-rules pass with zero-critique gate before ship; one elevation step, one colored dot. |
| Scope creep from "wire in all the logic" | §6/§7 split: Library-scoped in SP2, other-page logic explicitly SP3. |
| Group-override re-link corrupts associations | Reuse the existing `save_group_overrides` (pure JSON side map); previews just re-read — no library mutation. |
